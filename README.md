# キャリア羅針盤 — 運用マニュアル

転職・キャリア系アフィリエイトサイトの土台一式です。**週1〜2時間の運用**を前提に設計しています。

## 構成

```
career-site/
├── build.py          # サイトジェネレーター(依存ライブラリなし)
├── articles/         # Markdown記事(ここに追記していく)
├── static/style.css  # デザイン
└── docs/             # ビルド成果物(これを公開する)
```

## 初回セットアップ(合計 2〜3時間・1回だけ)

### 1. 無料ホスティングに公開(30分)

**GitHub Pages(推奨)**
1. GitHubで新規リポジトリ作成
2. このフォルダ一式をpush
3. Settings → Pages → Source を「main / docs フォルダ」に設定
4. 数分で `https://ユーザー名.github.io/リポジトリ名/` で公開される

独自ドメイン(年1,500円程度)は審査通過率とSEOに有利なので、余裕があればお名前.com等で取得し、Pages設定でカスタムドメインを指定。

### 2. ASP登録(30分)

転職ジャンルで登録すべきASP(すべて無料):

| ASP | 特徴 |
|---|---|
| A8.net | 最大手。審査が緩く最初に登録。転職案件多数 |
| もしもアフィリエイト | W報酬制度あり |
| アクセストレード | 転職・金融系に強い |
| afb | 転職・人材案件の単価が高め |

**注意:審査には記事5本以上が目安。** 同梱の3記事に加えて2〜3本追加してから申請すると通過率が上がります。

### 3. アフィリエイトリンク差し替え(15分)

各記事内の `https://REPLACE-WITH-YOUR-ASP-LINK` を、ASPで発行された広告リンクに置き換えて再ビルド。

### 4. 法令対応(済/確認のみ)

- 記事内CTAに「PR」表記、フッターに広告掲載の明記済み(景表法ステマ規制対応)
- プライバシーポリシーページは公開前に1枚追加推奨

## 週次運用フロー(週1〜2時間)

```
① 記事を1本書く(Markdownで articles/ に追加)…60〜90分
② python3 build.py                            …数秒
③ git add -A && git commit -m "add article" && git push …1分
```

### 記事作成をClaude Codeで半自動化する

学習中のClaude Codeがそのまま活きます。プロンプト例:

> articles/ 内の既存記事とフロントマター書式を参考に、
> 「面接の逆質問」をテーマにした2,500字の記事Markdownを作成し、
> articles/ に保存して build.py を実行して。
> CTAボックス(@cta記法)を1つ、自然な文脈で入れること。

自分の実務経験(業界知識・M&A・キャリア観)を箇条書きで渡して肉付けさせると、独自性が出て検索評価も上がります。**完全自動の量産は品質・規約両面で逆効果**なので、最終チェックだけは自分で行うのが「週1〜2時間」の中身です。

## 記事の書式

```markdown
---
title: 記事タイトル
date: 2026-07-10
category: 転職戦略
description: 一覧ページと検索結果に出る説明文
---

本文。## で見出し、- でリスト、**太字**、[リンク](URL) が使えます。

@cta サービス名 | 紹介文 | アフィリエイトURL | ボタン文言
```

## 収益化の現実的な目安

- 0〜3か月:記事10本到達+ASP審査通過が目標(収益ほぼゼロが普通)
- 3〜6か月:検索流入が立ち上がり始める。月数千円〜
- 転職エージェント案件は1件5,000〜15,000円程度と高単価。月2〜3件の成約で月1〜3万円が最初のマイルストーン

焦らず「毎週1本、24週で24本」の積み上げが最短ルートです。
